The data matrix

What is this matrix?

Every row is one observation. Every column is one variable. Together they form the matrix X — the starting point for PCA.

\[\mathbf{X} \in \mathbb{R}^{n \times p}\]

PCA finds new directions in this p-dimensional space that capture the most variance. But first — look at your raw numbers.

Matrix X
How to read this

Notation

X — the full matrix, shape n×p

xij — value at row i, column j

xj — the j-th column (one variable across all observations)

Colour guide

Orange values — high relative to that column's range

Blue values — low relative to that column's range

Notice: some columns may have much larger numbers than others — that's why standardisation matters.

Why we can't use this matrix directly

PCA finds directions of maximum variance. If one column has values in the thousands and another in the single digits, PCA will be dominated by the large-scale column — not because it's more important, but because its numbers are bigger.

The fix is standardisation: subtract the mean and divide by the standard deviation for each column. Every column ends up with mean 0 and standard deviation 1.

\[z_{ij} = \frac{x_{ij} - \mu_j}{\sigma_j}\]
numpy equivalent 
import numpy as np

# X is your raw data matrix, shape (n, p)
X = np.array(data)

# Inspect the matrix
print(X.shape)   # (n, p)
print(X.mean(axis=0))  # per-column means
print(X.std(axis=0))   # per-column std devs
3

Standardisation

Putting every variable on equal footing

For each column j, we subtract its mean and divide by its standard deviation:

\[z_{ij} = \frac{x_{ij} - \mu_j}{\sigma_j}\]

After this transform, every column has mean = 0 and std = 1. PCA can then measure variance fairly across all features.

Before vs after — first rows

Orange = more than 1σ above mean  ·  Blue = more than 1σ below mean  ·  all values now dimensionless

What we have now

Matrix Z — where every column has mean 0 and standard deviation 1. The data is now scale-invariant.

Next: we compute the covariance matrix of Z — a p×p matrix that captures how much every pair of variables moves together. It's the heart of PCA.

\[\Sigma = \frac{1}{n-1}\, Z^\top Z\]
numpy equivalent 
import numpy as np

# X is your raw data matrix, shape (n, p)
mu    = X.mean(axis=0)          # shape (p,)
sigma = X.std(axis=0, ddof=1)   # sample std, shape (p,)

Z = (X - mu) / sigma             # shape (n, p), mean=0 std=1 per column

# Verify
print(Z.mean(axis=0))            # → [~0, ~0, ...]
print(Z.std(axis=0, ddof=1))     # → [1.0, 1.0, ...]
4

Covariance matrix

What is a covariance matrix?

Each entry Cij measures how much variables i and j vary together. If large values of variable i tend to appear with large values of variable j, their covariance is positive. If they move in opposite directions, it's negative.

\[C = \frac{1}{n-1}\, Z^\top Z\]

Because we standardised first, the diagonal entries are all exactly 1 (variance of each standardised variable). The off-diagonal entries are the Pearson correlations between pairs of variables.

PCA will decompose this matrix into eigenvectors (principal directions) and eigenvalues (how much variance each direction explains).

Σ — covariance matrix (hover to inspect)
negative covariance
positive covariance  ·  ■ diagonal = variance (= 1 after standardisation)
How to read each cell
Negative covariance
When one variable goes up, the other tends to go down. They move in opposite directions across observations.
Near-zero covariance
The two variables are roughly independent — knowing one tells you little about the other.
Positive covariance
When one variable goes up, the other tends to also go up. They carry overlapping information.

Key properties of this matrix

Symmetric — C[i,j] always equals C[j,i]. The covariance between height and weight is the same as between weight and height.

Positive semi-definite — all eigenvalues will be ≥ 0. This is why PCA always produces real, non-negative variance explained values.

Diagonal = 1 — because we standardised. Without this, PCA would be dominated by whichever variable happens to have the largest scale.

numpy equivalent 
# Z is the standardised matrix, shape (n, p)
C = np.cov(Z.T)              # shape (p, p) — equivalent to Z.T @ Z / (n-1)

# C is symmetric: C[i,j] == C[j,i]
# Diagonal entries are all 1.0 (because Z is standardised)
print(np.diag(C))             # → [1.0, 1.0, ...]

Eigen decomposition

What comes next

We decompose the covariance matrix C into its eigenvectors and eigenvalues:

\[C = V \Lambda V^\top\]

Each eigenvector vk defines a principal component — a direction in your original p-dimensional space. The corresponding eigenvalue λk tells you how much variance is explained in that direction.

The first principal component is the direction of greatest variance. The second is perpendicular to the first, and explains the next most variance — and so on. Together, a handful of components often capture the vast majority of the structure in your data.
