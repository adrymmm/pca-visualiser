The data matrix

The data matrix X

Each row is one observation; each column is one variable. PCA finds directions in this p-dimensional space along which the data has maximum variance.

\[\mathbf{X} \in \mathbb{R}^{n \times p}\]
Matrix X
Intuition

Think of this as a spreadsheet: rows are observations (countries, patients, experiments) and columns are things measured about them. PCA's job is to find which combinations of columns tend to move together, and to express those patterns as new, simpler axes.

Notation

Notation

X: the full data matrix, shape n×p

xij: value at row i, column j

xj: the j-th column, one variable across all observations

Colour guide

Orange: high relative to that column's range

Blue: low relative to that column's range

Columns with different scales will distort PCA; standardisation addresses this.

Why standardisation is required

PCA finds directions of maximum variance. If one column has values in the thousands and another in the single digits, PCA will be dominated by the large-scale column, not because it carries more information, but simply because its numerical range is larger.

The fix is standardisation: subtract the mean and divide by the standard deviation for each column. Every column ends up with mean 0 and standard deviation 1.

\[z_{ij} = \frac{x_{ij} - \mu_j}{\sigma_j}\]
numpy equivalent 
import numpy as np

# X is your raw data matrix, shape (n, p)
X = np.array(data)

# Inspect the matrix
print(X.shape)   # (n, p)
print(X.mean(axis=0))  # per-column means
print(X.std(axis=0))   # per-column std devs

Standardisation

Centering and scaling each variable

For each column j, we subtract its mean and divide by its standard deviation:

\[z_{ij} = \frac{x_{ij} - \mu_j}{\sigma_j}\]

After this transform, every column has mean = 0 and std = 1. PCA can then measure variance fairly across all features.

Intuition

Suppose one column records annual salaries (tens of thousands) and another records age (tens). The salary column will appear to vary far more, purely because its numbers are larger. Standardisation converts every column into units of standard deviations above or below its mean, so PCA treats all variables on equal footing regardless of their original scale.

Before and after: first rows

Orange = more than 1σ above mean  ·  Blue = more than 1σ below mean  ·  all values now dimensionless

The standardised matrix Z

Every column of Z has mean 0 and standard deviation 1. The data is now scale-invariant, and PCA can measure variance fairly across all variables.

The next step computes the covariance matrix of Z: a p×p matrix recording how much each pair of variables moves together.

\[\Sigma = \frac{1}{n-1}\, Z^\top Z\]
numpy equivalent 
import numpy as np

# X is your raw data matrix, shape (n, p)
mu    = X.mean(axis=0)          # shape (p,)
sigma = X.std(axis=0, ddof=1)   # sample std, shape (p,)

Z = (X - mu) / sigma             # shape (n, p), mean=0 std=1 per column

# Verify
print(Z.mean(axis=0))            # → [~0, ~0, ...]
print(Z.std(axis=0, ddof=1))     # → [1.0, 1.0, ...]

Covariance matrix

What is a covariance matrix?

Each entry Cij measures how much variables i and j vary together. If large values of variable i tend to appear with large values of variable j, their covariance is positive. If they move in opposite directions, it's negative.

\[C = \frac{1}{n-1}\, Z^\top Z\]

Because we standardised first, the diagonal entries are all exactly 1 (variance of each standardised variable). The off-diagonal entries are the Pearson correlations between pairs of variables.

PCA will decompose this matrix into eigenvectors (principal directions) and eigenvalues (how much variance each direction explains).

Intuition

This matrix records which pairs of measurements rise and fall together. Tall people tend to weigh more, so height and weight have positive covariance. Knowing these co-movements tells PCA where the interesting structure lies: directions that combine several correlated variables carry more information than any single variable alone.

C: covariance matrix (hover to inspect)
negative covariance
positive covariance  ·  ■ diagonal: variance (= 1 after standardisation)
How to read each cell
Negative covariance
When one variable goes up, the other tends to go down. They move in opposite directions across observations.
Near-zero covariance
The two variables are roughly independent; knowing one tells you little about the other.
Positive covariance
When one variable goes up, the other tends to also go up. They carry overlapping information.

Key properties of this matrix

Symmetric: C[i,j] = C[j,i] for all i, j. The covariance between any two variables is the same in either direction.

Positive semi-definite: all eigenvalues are ≥ 0, which guarantees that variance explained values are real and non-negative.

Diagonal = 1: a consequence of standardisation. Without it, PCA would be dominated by whichever variable has the largest numerical range.

numpy equivalent 
# Z is the standardised matrix, shape (n, p)
C = np.cov(Z.T)              # shape (p, p) — equivalent to Z.T @ Z / (n-1)

# C is symmetric: C[i,j] == C[j,i]
# Diagonal entries are all 1.0 (because Z is standardised)
print(np.diag(C))             # → [1.0, 1.0, ...]

Eigen-decomposition

Spectral decomposition of C

The covariance matrix C is decomposed into its eigenvectors and eigenvalues:

\[C = V \Lambda V^\top\]

Each eigenvector vk defines a principal component: a direction in the original p-dimensional feature space. The corresponding eigenvalue λk gives the variance of the data projected onto that direction.

The eigenvectors are orthonormal and ordered by descending eigenvalue, so the first principal component is the direction of greatest variance, the second is orthogonal to it with the next greatest variance, and so on.

Intuition

Imagine casting a shadow of a cloud of points onto a wall by shining a torch from different angles. There is one angle where the shadow spreads out as much as possible: that direction is the first principal component. The second is perpendicular to the first and gives the next largest shadow. The eigenvalues measure how large each shadow is; the eigenvectors record which direction the torch was pointing.

Summary

The complete pipeline

Starting from a raw data matrix, PCA proceeds through four operations:

  1. Standardise each column to mean 0, standard deviation 1: \(z_{ij} = (x_{ij} - \mu_j)/\sigma_j\)
  2. Compute the covariance matrix of the standardised data: \(C = \tfrac{1}{n-1}Z^\top Z\)
  3. Eigen-decompose C into its eigenvectors and eigenvalues: \(C = V\Lambda V^\top\)
  4. Project onto the top k eigenvectors: \(X_\text{pca} = ZW\) where W contains the first k columns of V

The result is a lower-dimensional representation that retains as much variance as possible given the chosen k.

Intuition

The whole procedure is essentially finding the best angle from which to photograph a high-dimensional cloud of points, so that the resulting flat image loses as little information as possible.
